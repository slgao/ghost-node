# Using the VPN from China (GFW Bypass Guide)

## Overview

China's Great Firewall (GFW) uses deep packet inspection (DPI), IP blocking, DNS poisoning, and active probing to detect and block VPN traffic. This guide explains which protocols work best and how to connect from inside China.

---

## Protocol Comparison

| Protocol | Detection Resistance | Speed | Recommendation |
|----------|---------------------|-------|----------------|
| **VLESS + REALITY** | ★★★★★ Mimics real TLS 1.3 to microsoft.com | Fast (TCP) | **Best choice** — use first |
| **VLESS + WS + TLS** | ★★★★☆ Works behind Cloudflare CDN | Medium | Use if REALITY blocked |
| **VLESS + gRPC + TLS** | ★★★★☆ HTTP/2 multiplexing | Medium | CDN fallback |
| **Hysteria2** | ★★★☆☆ QUIC/UDP — easily rate-limited | Very fast | Use for high-bandwidth, may be blocked |

**Recommended stack**: Deploy REALITY on port 443 first. If blocked, fall back to WS+TLS behind Cloudflare CDN.

## Step 1: Deploy a Server

Run the setup script on a **non-Chinese VPS** (Hong Kong, Japan, Singapore, US, Europe):

```bash
ssh root@YOUR_SERVER_IP

# Download and run the setup script
curl -sSL https://raw.githubusercontent.com/your-org/vpn/main/scripts/setup-server.sh | \
  CONTROL_PLANE=https://api.yourdomain.com \
  ADMIN_TOKEN=<your-admin-jwt> \
  bash
```

The script will:
1. Install and configure Xray with VLESS+REALITY
2. Generate a unique UUID, X25519 key pair, and short ID
3. Register the node with the control plane
4. Print a `vless://` URI you can import directly

**Recommended server locations** (ranked by latency from mainland China):
- Hong Kong (lowest latency, ~10–30ms)
- Japan / Singapore (~50–100ms)
- US West Coast (~150–200ms)

## Step 2: Get Your Client Config

### Option A — Import via subscription URL (recommended)

After logging into the platform and getting a JWT token:

```bash
# On your laptop/phone before entering China — save the configs locally
./scripts/gen-client-config.sh \
  --api   https://api.yourdomain.com \
  --token YOUR_JWT_TOKEN \
  --node  YOUR_NODE_UUID \
  --out   ~/vpn-configs
```

This generates:
- `~/vpn-configs/vless.txt` — base64 URI list for v2rayN, NekoRay
- `~/vpn-configs/clash.yaml` — Clash Meta / Stash / Clash Verge
- `~/vpn-configs/singbox.json` — sing-box

### Option B — Subscription URL (auto-updates)

Most clients support a subscription URL that auto-refreshes configs:

| Format | URL |
|--------|-----|
| VLESS (v2rayN) | `https://api.yourdomain.com/api/v1/nodes/NODE_ID/subscription?format=vless` |
| Clash Meta | `https://api.yourdomain.com/api/v1/nodes/NODE_ID/subscription?format=clash` |
| sing-box | `https://api.yourdomain.com/api/v1/nodes/NODE_ID/subscription?format=singbox` |

> **Important**: The subscription endpoint requires a JWT Bearer token in the `Authorization` header. Some clients (Shadowrocket, Quantumult X) support custom headers for subscription URLs.

## Troubleshooting from China

### "Connection refused" or timeout
- The port may be blocked by your ISP. Try port 443 (already the default for REALITY).
- Switch from REALITY to VLESS+WS+TLS via Cloudflare (CDN hides your server IP).

### Speed is very slow
- Try a Hong Kong or Japan server instead of US/EU.
- Enable Hysteria2 for high-bandwidth use (UDP may be less throttled on some ISPs).

### Works on Wi-Fi but not mobile data
- Mobile operators (China Mobile, Unicom, Telecom) have different routing. HK servers work best on China Telecom CN2 routes.

### REALITY stopped working
- The camouflage domain (`www.microsoft.com`) must support TLS 1.3 + X25519. If Microsoft changes this, update `realitySettings.dest` and regenerate keys.
- Generate new keys: `xray x25519` and `openssl rand -hex 8`

### How to verify the VPN is active

```bash
# Your IP should show the server's country, not China
curl https://ipapi.co/json | python3 -m json.tool

# Or visit: https://ip.sb or https://whoer.net
```

## Using the CDN Fallback (VLESS + WebSocket + Cloudflare)

If your server IP gets blocked, hide it behind Cloudflare:

1. Point a subdomain (`cdn.yourdomain.com`) to your server IP in Cloudflare DNS
2. Enable **Cloudflare Proxy** (orange cloud) — Cloudflare's IPs are rarely blocked
3. Use the `xray-server-ws-tls.json` config on the server (port 8443)
4. Set the VLESS client host to `cdn.yourdomain.com` with WebSocket path `/ws`
5. Use port 443 or 80 (Cloudflare only proxies specific ports)

Cloudflare-compatible ports: `80, 443, 2052, 2053, 2082, 2083, 2086, 2087, 2095, 2096, 8080, 8443, 8880`

## Automatic Failover

The VPN platform's transport manager automatically fails over to the next healthy provider when latency exceeds thresholds or health checks fail. You don't need to manually switch — just ensure multiple transport profiles are registered for your node via the admin API:

```bash
# Add a WS+TLS profile as fallback
curl -X POST https://api.yourdomain.com/api/v1/admin/nodes/NODE_ID/transports \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "xray",
    "port": 8443,
    "priority": 5,
    "config": {
      "protocol": "vless",
      "transport": "ws",
      "uuid": "YOUR_UUID",
      "host": "cdn.yourdomain.com",
      "path": "/ws",
      "sni": "cdn.yourdomain.com"
    }
  }'
```

Lower `priority` = used as fallback. The manager selects the active profile with the lowest latency.

## Security Notes

- **Never reuse UUIDs** across servers. Each node should have a unique UUID.
- **Keep the REALITY private key secret** — it's stored in `/root/vpn-server-credentials.env` (chmod 600).
- **Rotate credentials periodically** via the admin API: re-run the setup script or update the transport profile.
- **The subscription endpoint requires authentication** — JWT tokens expire after 15 minutes; use refresh tokens for long-lived access.
